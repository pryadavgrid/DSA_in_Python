# 📘 Data Structure and Algorithm in Python

## 📌 Introduction

Data Structure and Algorithm (DSA) is very important in programming.
It helps us to write **fast**, **efficient**, and **clean** code.

If you want to become a good Python developer, you must understand DSA.

---

# 🔹 Basic Definitions

## 1️⃣ What is Data?

**Data** means information.

Example:

* Numbers → `10, 20, 30`
* Name → `"Prateek"`
* List → `[1, 2, 3]`

Data is the raw value that we store and use in a program.

---

## 2️⃣ What is Structure?

**Structure** means the way of organizing data.

Example:

* List stores data in order.
* Dictionary stores data in key-value format.

Structure decides **how data is arranged in memory**.

---

## 3️⃣ What is Algorithm?

An **Algorithm** is a step-by-step process to solve a problem.

Example:

* Steps to find the largest number in a list.
* Steps to sort numbers in ascending order.

Simple meaning:
👉 Algorithm = Logic + Steps to solve a problem.

---

## 4️⃣ What is Data Structure and Algorithm?

* **Data Structure** = How data is stored.
* **Algorithm** = How data is processed.

Both work together.

Example:
If data is stored in a list (Data Structure),
then sorting that list (Algorithm) makes it useful.

---

# 📚 Types of Data Structures

Data Structures are mainly divided into:

1. Linear Data Structure
2. Non-Linear Data Structure

---

# 🔹 Linear Data Structures

Linear means data is arranged in a straight line.

---

## 1️⃣ Linked List

A **Linked List** is a collection of nodes.

Each node has:

* Data
* Address of next node

Types:

* Singly Linked List
* Doubly Linked List
* Circular Linked List

Why use it?

* Easy insertion and deletion.
* No fixed size.

---

## 2️⃣ Stack

Stack follows:

👉 **LIFO (Last In First Out)**

Example:

* Stack of plates.

Operations:

* Push (Add element)
* Pop (Remove element)
* Peek (See top element)

Uses:

* Undo operation
* Function calls
* Expression evaluation

---

## 3️⃣ Queue

Queue follows:

👉 **FIFO (First In First Out)**

Example:

* Line in a bank.

Operations:

* Enqueue (Add)
* Dequeue (Remove)

Types:

* Simple Queue
* Circular Queue
* Priority Queue
* Deque (Double Ended Queue)

---

# 🔹 Non-Linear Data Structures

Non-linear means data is not in straight line.

---

## 4️⃣ Binary Tree

A **Binary Tree** is a tree where:

Each node has at most 2 children:

* Left child
* Right child

Terms:

* Root
* Leaf
* Parent
* Child
* Height
* Depth

Uses:

* Hierarchical data
* Expression trees

---

## 5️⃣ Binary Search Tree (BST)

A special type of Binary Tree.

Rule:

* Left child < Parent
* Right child > Parent

Why important?

* Searching is faster than normal tree.
* Average time complexity: O(log n)

---

## 6️⃣ Heap

Heap is a special Binary Tree.

Types:

* Min Heap → Smallest element at top
* Max Heap → Largest element at top

Used in:

* Priority Queue
* Heap Sort
* Finding smallest/largest element quickly

Time complexity:

* Insert → O(log n)
* Delete → O(log n)

---

# 📊 Algorithms in DSA

---

## 7️⃣ Sorting

Sorting means arranging data in order.

Types of Sorting:

* Bubble Sort
* Selection Sort
* Insertion Sort
* Merge Sort
* Quick Sort
* Heap Sort

Why sorting is important?

* Makes searching faster.
* Easy to understand data.

---

## 8️⃣ Searching

Searching means finding an element.

Types:

### 1. Linear Search

* Check one by one.
* Time Complexity → O(n)

### 2. Binary Search

* Works only on sorted data.
* Divide data into half.
* Time Complexity → O(log n)

Binary Search is much faster than Linear Search.

---

## 9️⃣ Hashing

Hashing is a technique to store data using a key.

It uses a **Hash Function** to convert key into index.

Example:
Python Dictionary uses Hashing.

Advantages:

* Very fast searching → O(1) average
* Fast insertion and deletion
